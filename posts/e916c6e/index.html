多线程进阶(一)

一. 条件变量

重复加锁, 影响效率, 可通过双重锁定的方式, 减少加锁, 提升效率

bool IsoutMsg()
{
    // 双重锁定, 提升效率
    if (!msgRecvQueue.empty())
    {
        unique_lock<mutex> ulock1(mtx1);
        unique_lock<mutex> ulock2(mtx2);
        if (!msgRecvQueue.empty())
        {
            cout << "删除元素 " << msgRecvQueue.front() << endl;
            msgRecvQueue.pop_front();
            mtx1.unlock();
            mtx2.unlock();
            return true;
        }
    }
    return false;
}

本方法仍不够完美, 以下给出更优的解决办法

1. condition_variable

std::condition_variable实际上是一个类，是一个和条件相关的类，说白了就是等待一个条件达成。

void outMsg()
{
    int command = 0;
    int time = 0;
    for(int i = 0; i < 20; i++)
    {
        unique_lock<mutex> ulock1(mtx1);
        // 如果第二个参数lambda表达式返回值是false, 那么wait()将解锁互斥量, 并堵塞本行
        // 直到其他线程调用notify_one()为止
        my_cond.wait(ulock1, [this]
                     {
                         if (!msgRecvQueue.empty()) return true;
                         else return false;
                     });

        command = msgRecvQueue.front();
        msgRecvQueue.pop_front();
        ulock1.unlock();
        cout << "删除元素 " << command << endl;
    }
    cout << "END" << endl;
}

wait()用来等一个东西

如果第二个参数的lambda表达式返回值是false，那么wait()将解锁互斥量，并阻塞到本行
如果第二个参数的lambda表达式返回值是true，那么wait()直接返回并继续执行。

阻塞到什么时候为止呢？阻塞到其他某个线程调用notify_one()成员函数为止；

如果没有第二个参数，那么效果跟第二个参数lambda表达式返回false效果一样

wait()将解锁互斥量，并阻塞到本行，阻塞到其他某个线程调用notify_one()成员函数为止。

当其他线程用notify_one()将本线程wait()唤醒后，这个wait恢复后

  1. wait()不断尝试获取互斥量锁，如果获取不到那么流程就卡在wait()这里等待获取，如果获取到了，那么wait()就继续执行，获取到了锁

  2. 如果wait有第二个参数就判断这个lambda表达式。

    a)如果表达式为false，那wait又对互斥量解锁，然后又休眠，等待再次被notify_one()唤醒
    b)如果lambda表达式为true，则wait返回，流程可以继续执行（此时互斥量已被锁住）。

  3. 如果wait没有第二个参数，则wait返回，流程走下去。

流程只要走到了wait()下面则互斥量一定被锁住了。

完整代码:

#include <iostream>
#include <thread>
#include <vector>
#include <list>
#include <queue>
#include <mutex>
#include <condition_variable>

using namespace std;

class A
{
public:
    void getMsg()
    {
        for (int i = 0; i < 20; i++)
        {
            cout << "Msg " << i << " got." << endl;

            unique_lock<mutex> myGuard(mtx1, adopt_lock);
//            lock(mtx1, mtx2);
            msgRecvQueue.push_back(i);
            my_cond.notify_one();
//            mtx1.unlock();
//            mtx2.unlock();
        }
    }

    void outMsg()
    {
        int command = 0;
        int time = 0;
        for(int i = 0; i < 20; i++)
        {
            unique_lock<mutex> ulock1(mtx1);
            // 如果第二个参数lambda表达式返回值是false, 那么wait()将解锁互斥量, 并堵塞本行
            // 直到其他线程调用notify_one()为止
            my_cond.wait(ulock1, [this]
            {
                if (!msgRecvQueue.empty()) return true;
                else return false;
            });

            command = msgRecvQueue.front();
            msgRecvQueue.pop_front();
            ulock1.unlock();
            cout << "删除元素 " << command << endl;
        }
        cout << "END" << endl;
    }


private:
    list<int> msgRecvQueue;
    mutex mtx1;
    mutex mtx2;
    condition_variable my_cond;  // 生成一个条件变量对象
};


int main()
{
    A my_a;
    thread ts1(&A::outMsg, &my_a);
    thread ts2(&A::getMsg, &my_a);
    ts1.join();
    ts2.join();

    return 0;
}

二、深入思考

上面的代码可能导致出现一种情况：
因为outMsg()与getMsg()并不是一对一执行的，所以当程序循环执行很多次以后，可能在msgRecvQueue 中已经有了很多消息，但是，outMsg()还是被唤醒一次只处理一条数据。这时可以考虑把outMsg()多执行几次，或者对getMsg()进行限流。

notify_all()

notify_one()：通知一个线程的wait()

notify_all()：通知所有线程的wait()

二. 创建后台任务并返回值–future类

std::async std::future

std::async是一个函数模板，用来启动一个异步任务，启动起来一个异步任务之后，它返回一个std::future对象，这个对象是个类模板。

异步任务的定义:

启动一个异步任务就是自动创建一个线程，并开始 执行对应的线程入口函数，它返回一个std::future对象，这个std::future对象中就含有线程入口函数所返回的结果，我们可以通过调用future对象的成员函数get()来获取结果。

“future”将来的意思，也有人称呼std::future提供了一种访问异步操作结果的机制，就是说这个结果你可能没办法马上拿到，但是在不久的将来，这个线程执行完毕的时候，你就能够拿到结果了，所以，大家这么理解：future中保存着一个值，这个值是在将来的某个时刻能够拿到。

  • std::future对象的get()成员函数会等待线程执行结束并返回结果，拿不到结果它就会一直等待，感觉有点像join()。但是，它是可以获取结果的。

  • std::future对象的wait()成员函数，用于等待线程返回，本身并不返回结果，这个效果和 std::thread 的join()更像。

使用get()返回:

#include <iostream>
#include <thread>
#include <mutex>
#include <future>
using namespace std;

int mythread()
{
    cout << "mythread started" << "\tthread id: " << this_thread::get_id() << endl;
    chrono::milliseconds dura(5000); // 5s
    this_thread::sleep_for(dura);
    cout << "mythread finished" << "\tthread id: " << this_thread::get_id() << endl;
    return 5;
}


int main()
{
    cout << "main thread started" << "\tthread id: " << this_thread::get_id() << endl;
    future<int> res = async(mythread);
    // get()获取返回值
    cout << res.get() << endl;

    cout << "end" << endl;

    return 0;
}

get()只能调用一次!!!!

因为get()函数的设计是一个移动语义，相当于将result中的值移动到了a中，再次get就报告了异常。

std::launch类型（枚举类型）

我们通过向std::async()传递一个参数，该参数是std::launch类型（枚举类型），来达到一些特殊的目的：

  1. std::lunch::deferred：
    （defer推迟，延期）表示线程入口函数的调用会被延迟，一直到std::future的wait()或者get()函数被调用时（由主线程调用）才会执行；如果wait()或者get()没有被调用，则不会执行。
    实际上根本就没有创建新线程。std::launch::deferred意思时延迟调用，并没有创建新线程，是在主线程中调用的线程入口函数。

  1. std::launch::async:

std::packaged_task

std::packaged_task：打包任务，把任务包装起来。

类模板，它的模板参数是各种可调用对象，通过packaged_task把各种可调用对象包装起来，方便将来作为线程入口函数来调用。

int main()
{
    int tmp = 230;
    cout << "main thread started" << "\tthread id: " << this_thread::get_id() << endl;
    packaged_task<int(int)> mypt(mythread);
    thread t1(ref(mypt), tmp);
    t1.join();
    /// res 保存mypt将要返回的值
    future<int> res = mypt.get_future();
    cout << "continue..." << endl;
    /// get()获取返回值
    cout << res.get() << endl;

    /// wait()不返回结果
//    res.wait();

    cout << "end" << endl;

    return 0;
}

packaged_task包装起来的可调用对象还可以直接调用，从这个角度来讲，packaged_task对象也是一个可调用对象

lambda的直接调用:

int main()

    return 0;
}

std::promise

std::promise，类模板
我们能够在某个线程中给它赋值，然后我们可以在其他线程中，把这个值取出来

#include <thread>
#include <iostream>
#include <future>
using namespace std;
 
void mythread(std::promise<int> &tmp, int clac) {
    cout << "mythread() start" << "threadid = " << std::this_thread::get_id() << endl;
    std::chrono::milliseconds dura(5000);
    std::this_thread::sleep_for(dura);
    cout << "mythread() end" << "threadid = " << std::this_thread::get_id() << endl;
    int result = clac;
    tmp.set_value(result); //结果保存到了tmp这个对象中
    return;
}
 
vector<std::packaged_task<int(int)>> task_vec;
 
int main() {
    std::promise<int> myprom;
    std::thread t1(mythread, std::ref(myprom), 180);
    t1.join(); //在这里线程已经执行完了
    std::future<int> fu1 = myprom.get_future(); //promise和future绑定，用于获取线程返回值
    auto result = fu1.get();
    cout << "result = " << result << endl;
}

三. std::future 深入理解

future其他成员函数

  1. std::future_status status = result.wait_for(std::chrono::seconds(几秒));

卡住当前流程，等待std::async()的异步任务运行一段时间，然后返回其状态std::future_status。

  • 如果std::async()的参数是std::launch::deferred（延迟执行），则不会卡住主流程。
  • std::future_status是枚举类型，表示异步任务的执行状态。类型的取值有:
    • std::future_status::timeout
    • std::future_status::ready
    • std::future_status::deferred

代码示例:

#include <iostream>
#include <thread>
#include <mutex>
#include <future>
using namespace std;

/**
  * @brief 线程入口函数
  * @param int类型 num 数字
  * @retval 5
  */
int mythread(int num)
{
    cout << "num: " << num << endl;
    cout << "mythread started" << "\tthread id: " << this_thread::get_id() << endl;
    chrono::milliseconds dura(5000); // 5s
    this_thread::sleep_for(dura);
    cout << "mythread finished" << "\tthread id: " << this_thread::get_id() << endl;
    return 5;
}

int main()
{
    int tmp = 230;
    cout << "main thread started" << "\tthread id: " << this_thread::get_id() << endl;
    future<int> res = async(/*launch::deferred,*/ mythread, tmp);
    cout << "continue..." << endl;

    /// 枚举类型
    future_status status = res.wait_for(chrono::seconds(6));
    if (status == future_status::timeout)
    {
        /// 超时了
        cout << "timeout!!" << endl;
    }
    else if (status == future_status::ready)
    {
        /// 成功返回
        cout << "线程成功执行!!" << endl;
    }
    else if (status == future_status::deferred)
    {
        /// 延迟执行
        cout << "线程延迟执行!!!" << endl;
    }

    cout << "end" << endl;

    return 0;
}

std::shared_future

  • std::future的 get() 成员函数是转移数据

  • std::shared_future 的 get()成员函数是复制数据

即shared_future的get()可以重复调用

#include <iostream>
#include <thread>
#include <mutex>
#include <future>
using namespace std;

/**
  * @brief 线程入口函数
  * @param int类型 num 数字
  * @retval 5
  */
int mythread()
{
    cout << "mythread started" << "\tthread id: " << this_thread::get_id() << endl;
    chrono::milliseconds dura(5000); // 5s
    this_thread::sleep_for(dura);
    cout << "mythread finished" << "\tthread id: " << this_thread::get_id() << endl;
    return 5;
}

int main()
{
    cout << "main thread started" << "\tthread id: " << this_thread::get_id() << endl;
    packaged_task<int()> mypt(mythread);
    thread t1(ref(mypt));
    future<int> res_1 = mypt.get_future();

    /// 判断future中的值是不是一个有效值
    bool flag_get = res_1.valid();

    ///执行完毕后result_s里有值，而result里空了

    ///std::shared_future<int> result_s(std::move(result));
    ///通过get_future返回值直接构造一个shared_future对象
    ///std::shared_future<int> result_s(mypt.get_future());

    shared_future<int> result_s(res_1.share());
    t1.join();

    auto myres1 = result_s.get();
    auto myres2 = result_s.get();

    cout << "myres1 = " << myres1 << " myres2 = " << myres2 << endl;
    cout << "end" << endl;

    return 0;
}
